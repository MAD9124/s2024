# The brief

This is the third of three take home assignments related to building a backend web service to support a simple golf application called golfR.

In the previous assignments you built the base for the golfR RESTful API using Node.js, the Express framework and MongoDB. For this assignment you will enhance that foundation with Google OAuth 2.0 authentication.

In addition to correctly implementing all of the requirements from the previous assignment. The application will implement the following.

# Course

A golf course is the location where a round of golf is played. Traditionally, a golf course will have 18 holes of varying length and difficulty. A par is assigned to each hole (either 3, 4 or 5) and represents the score a good golfer will score on the hole, allowing for 2 putts.

For this API, we will want to know the name of the course, and the par and distance of each of the 18 holes.

Property Type Required Min Length Max Length

name String true 3 250
holes [Hole] true 18 18

# Hole

Property Type Required Min Value Max Value
par Number true 3 5
distance Number true 0 999

Do not create a collection for Hole!

Hole will be a nested object that only lives in the Course collection. It should not have an _id property

  1. All requests to the round resource are only available to authenticated users. All requests to the course resource are available to all users, authenticated or unauthenticated.

  2. All requests to the round resource should be limited to the authenticated user of the given round. This means the get / request should return a list of only the authenticated user's rounds. Any request with an id should return status 403 if the round does not belong to the authenticated user.

  3. Ensure all expected errors are properly formatted, and respond with the correct status code. This includes malformed object ids and failing mongoose validation errors.All errors messages should end with 2 periods

  4. Use security best practices to guard again Cross Site Scripting (XSS) and database injection attacks.

  5. Ensure that you write clean and readable code. Pay attention to:

  • no runtime errors
  • consistent 2 space indentation
  • USE YARN, not NPM
  • follow the project structure used in this course
  • properly formatted JSON responses with either data or error at the root
  • no debugging console logs
  • no commented out code
  • semantically descriptive names for variables and functions
  • use a .gitignore file and do not commit node_modules/ or .env files
  • include .env.example file

    • correct project name
