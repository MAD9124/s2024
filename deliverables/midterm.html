# Joint Project

The midterm project will be a two part joint assignment between MAD9124 and MAD9022. You will build a complete full-stack solution. The web service API portion will be graded for MAD9124, and the front-end application functionality, design, and usability will be graded for MAD9022.

# The Brief

Using Node.js, and Express, you will build the web service for a prototype of a fictitious application for tracking movies. This initial proof of concept (PoC) will have limited capabilities but should demonstrate the potential of a full-scale application.

# Core Requirements

  1. Using the Express.js framework, the API will follow these user stories:

  • A user can do a text search for movies that returns a limited response, designed to populated a search page.

  • A user can sort this response either by popularity, release date, or by votes.

  • A user can lookup a movie by id to get a more detailed response.

  • A user can favourite a movie by id.

  • A user can view a list of favourited movies.

  • Find the starter dataset here (opens new window)

  1. The routes will be as follows:

  • The 3 text search routes:

    • GET /api/movies?keyword=[text search]&sort=popularity: The searched for movies sorted by popularity

    • GET /api/movies?keyword=[text search]&sort=release-date: The searched for movies sorted by release date

    • GET /api/movies?keyword=[text search]&sort=vote: The searched for movies sorted by votes

      • Each route should return a filtered list where the title matches the keyword.

        • hint movie.title.includes(keyword)

      • It should return the movies in descending order based on the sort value

        • popularity -> popularity
        • release-date -> release_date
        • vote -> vote_average

      • Maximum 10 movies should be returned

      • Each of these should have the following format:

      {
  "id": "Number",
  "title": "String",
  "imageUrl": "String" // NOTE - this should be a smaller image size
}

  • /api/movies/:id: The detailed response.

    • Should return the full response from the array.

    • Should return an imageUrl property with a larger image format.

  • /api/movies/favourites: Returns a list of favourite movies.

    • Should have the same format as the main getAll routes

  • POST/api/movies/:id/favourite: Add the movie to the favourites list

    • Should not allow duplicates.

      • If the movie id being favourited is all ready included in the list, respond the same way, but don't add it a second time to the list.

    • Should return the favourited movie in the same format as getOne

  • POST /api/movies:

    • Not required

  • PUT, PATCH & DELETE /api/movies/:id:

    • Not required

Order Matters!

Make sure to place the GET /favourites route above GET /:id, or it will think the word favourites is an id!

CORS

We didn't have time to talk about CORS in class yet, but please follow these instructions to stop your server from sending CORS errors!

  1. Install cors
   npm i cors
  1. Import cors in src/index.js
const cors = require("cors");
  1. Add the following line to you code, after defining your app:
const app = express();
// add the following line:
app.use(cors());
